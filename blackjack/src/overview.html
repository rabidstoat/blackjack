This project contains a reference implementation for the Blackjack 1.0 protocol proposed for a CS544 class project. It includes both a basic server, and a simple command-line client.

As per assignment requirements, we have marked code that implements one of the five protocol requirements (STATEFUL, CONCURRENT, SERVICE, CLIENT, UI) in both class and method javadoc-style comments, but also at places in inline comments in the code. Comments are identified with the keyword in all caps. Classes that are involved in each of the requirements are linked below:

  1. STATEFUL: Follow the state transitions as per the DFA, checking and validating on client and server side as appropriate

    • All concrete classes in the drexel.edu.blackjack.client.screens package
    • All classes in the drexel.edu.blackjack.client.screens package
    • All classes in the drexel.edu.blackjack.server.timeouts package
    • The drexel.edu.blackjack.server.BlackjackProtocol class

  2. CONCURRENT: The server must be able to handle multiple clients

    • The drexel.edu.blackjack.server.BlackjackProtocol class
    • The drexel.edu.blackjack.server.BlackjackServer class
    • The drexel.edu.blackjack.server.BlackjackServerThread class

  3. SERVICE: The server must bind to a hard-coded port and the client defaults to this port number

    • The drexel.edu.blackjack.server.BlackjackServer class
    • The drexel.edu.blackjack.client.BlackjackCLClient class

  4. CLIENT: The client must be able to specify the hostname or IP address of the server. This is not yet implemented, Duc will be working on it, but I am linking a class that's likely to be involved with it for now.

    • The drexel.edu.blackjack.client.BlackjackCLClient class

  5. UI: The protocol logic is in the client and not exposed to the user

    • The drexel.edu.blackjack.client.BlackjackCLClient class
    • All classes in the drexel.edu.blackjack.client.screens package
    • The drexel.edu.blackjack.cards.Card class
    • The drexel.edu.blackjack.cards.Hand class
    • The drexel.edu.blackjack.client.in.MessagesFromServerListener interface
    • The drexel.edu.blackjack.client.out.MessagesToServerListener interface
